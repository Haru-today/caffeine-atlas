Data flow

리포트 데이터가 누적되는 흐름

회원 계정이 생기면 리포트 결과, 유전자형, 설문, 추적 반응을 한 사람의 기록으로 연결할 수 있습니다.

1

회원가입

이메일 인증 또는 휴대폰 인증으로 사용자 계정을 만듭니다.

2

동의 관리

유전자 정보, 건강 설문, 연구 활용 여부에 대한 동의를 분리해 저장합니다.

3

리포트 저장

유전자형 입력값, 산출 결과, 리포트 버전을 사용자 계정에 연결합니다.

4

추적 설문

섭취량, 수면, 두근거림, 불안감 등 실제 반응을 반복 입력받습니다.

5

관리자 분석

개인정보를 분리한 뒤 집단 통계와 리포트 보정 자료로 활용합니다.

Database

권장 데이터 테이블

테이블 저장 내용 운영 포인트
users 회원 ID, 이메일, 이름, 가입일 비밀번호는 직접 저장하지 않고 해시 처리
consents 리포트 데이터 저장 동의, 연구 활용 동의, 철회 이력 동의 버전과 날짜를 반드시 보관
genotypes rs762551, rs2069514, AHR 관련 SNP 입력값 민감정보로 분류해 접근 권한 제한
reports 점수, 유형, 해석 문구, 생성 시각 결과 기준 버전을 함께 저장
diaries 카페인 섭취량, 시간, 수면, 증상 반복 입력으로 실제 반응 데이터 축적
audit_logs 로그인, 조회, 수정, 삭제 기록 관리자 접근까지 기록

Build plan

실제 구축 시 추천 방식

지금 사이트는 정적 프로토타입입니다. 데이터를 누적하려면 아래 중 하나로 백엔드를 붙이면 됩니다.

빠른 사업화

Next.js + Supabase

회원가입, 로그인, PostgreSQL, 파일 저장, 관리자 권한을 빠르게 붙일 수 있습니다. 초기 MVP에 가장 현실적입니다.

모바일 확장

Firebase Auth + Firestore

로그인과 실시간 데이터 저장이 빠릅니다. 앱 확장 계획이 있으면 적합합니다.

기관용 운영

서버 API + 전용 DB

기관 연동, 권한 분리, 감사 로그, 데이터 암호화 정책을 세밀하게 통제할 수 있습니다.
